## Scenario

A user submitted a ticket reporting that their PC's desktop had gone blank — no icons were visible, and the system clock/date was missing from the taskbar. As the assigned support specialist, I logged the ticket, documented the symptoms with the user's attached screenshot, escalated it to a teammate for hands-on investigation, tracked it through to root-cause resolution, and closed it out with a clear summary of the fix.

## Ticket Lifecycle

### 1. Ticket Intake — User-Reported Symptoms

The user submitted the following report through the service desk:

> "After logging into my PC, I noticed the desktop is blank, the icons are hidden, and there is no Windows date and time. Please check this incident. I am attaching a screenshot."

I logged the ticket with:

- **Status:** Open
- **Priority:** 1 (Low)
- **Attachment:** User-provided screenshot of the affected desktop

`[Insert Screenshot Here — Ticket creation with description and attachment]`

---

### 2. Documented the Reported Symptoms

Before assigning the ticket, I made sure the description clearly captured all three symptoms the user reported — the blank desktop, the hidden icons, and the missing taskbar clock — so whoever picked it up next would have full context without needing to re-contact the user.

---

### 3. Escalated the Ticket for Investigation

Since a deeper look at the machine was needed, I transferred the ticket to a teammate (Martin) with clear instructions to investigate and resolve:

> "Hi Martin, Investigate this ticket and fix this issue."

- **Status:** Open
- **Transferred To:** Martin

`[Insert Screenshot Here — Ticket transfer with assigned owner]`

---

### 4. Root Cause Identified

Investigation traced the missing icons and clock display back to an unwanted program running on the machine, which was interfering with normal desktop behavior.

---

### 5. Issue Resolved

The affected system was scanned, the interfering software was removed, and the antivirus software on the machine was updated to help prevent the same issue from recurring.

---

### 6. Ticket Closed With Resolution Notes

Once the fix was confirmed, I updated and closed the ticket with a clear, non-technical summary so the record was easy for anyone to reference later:

> "Hi Team, I have investigated the incident and this was due to a malware attack. I have scanned the user's system and resolved the malware that was found and updated the antivirus of the user's machine."

**Actions logged:**
- Removed the unwanted software
- Updated the antivirus software

## Best Practices Shared With the User

1. **Restart the PC after any desktop-display issue** to confirm the fix has fully taken effect
2. **Keep antivirus software active and up to date** to catch unwanted programs early
3. **Avoid downloading software from unverified sources**, a common cause of unexpected desktop behavior
4. **Attach screenshots when submitting a ticket** — they significantly speed up diagnosis
5. **Report display issues right away** rather than waiting, since some causes can affect other parts of the system over time

---

## What I Learned

This ticket reinforced how much a clear, well-documented ticket trail matters in service desk work. Capturing the user's exact symptoms up front, escalating with specific instructions, and closing with a plain-language summary of what was done meant the issue could be understood and verified by anyone reviewing the ticket afterward — not just the person who worked it.
